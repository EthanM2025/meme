## 2026-06-04 · Cross-nav extensions, multi-pet, partition resize, task-complete chimes

### English

#### Highlights
- Four screens now reachable from the main two-page view via a "cross" nav: swipe up → Dashboard, swipe down → Pet (three pets to choose from, left/right between them), inverse swipe returns to the originating main page.
- Audio playback works for the first time: a short tone fires through the on-board AW8737A speaker when Claude Code finishes a turn (880 Hz) and when Codex finishes a turn (660 Hz). The speaker amplifier is power-cycled per chime so there is no idle hiss between events.
- Global right-button gesture: single press = Enter, double press = Cmd+A + Delete (clear input).
- Device no longer enters idle-sleep while USB is plugged in.

#### New features

**Multi-pet (kitty / diana / nier)**
- Added two more pets sourced from petdex sprite library (#diana-2 「Diana」, #nier-2b 「2B」) alongside the existing kitten.
- Pet screens now form a left/right strip — kitty → diana → nier — bounded at both ends (no wrap). Up-swipe from any pet returns to the last main page (Claude or Codex).
- Pet state machine is generic (`PetSpec`) — adding a new pet is one entry in the `PETS[]` table.
- Each pet uses 96×104 RGB565A8 frames (downscaled 2× from petdex's 192×208) so the 3-pet bundle fits in ≈2.5 MB embedded data.
- Animation cadence: each pet sits in **idle** for 2–5 cycles (~5–13 s) then plays one random expression (wait / fail / review) for one cycle before returning to idle. Wave overrides everything while the record button is held.
- Frame durations were rescaled to 1.5× of the petdex defaults so motion feels calm, not jittery.

**Task-complete chimes**
- New module `mic-test/main/chime.{h,cpp}`: dedicated FreeRTOS task pulls from a 1-deep queue (drop newest-wins on overflow), generates sine-wave samples with a fast exponential decay + ~6 ms attack ramp, writes them to the ES8311 codec at full output volume.
- Server side (`mac-server/server.py` → `push_state_loop`) emits a one-off WS frame `{"type":"chime","agent":"claude"|"codex"}` when a new native task-complete signal is detected.
  - **Claude:** strictly-forward change in the timestamp of the most recent assistant message whose `stop_reason == "end_turn"` (read from `~/.claude/projects/*/*.jsonl`).
  - **Codex:** strictly-forward change in the most recent `event_msg.type == "task_complete"` row in `~/.codex/state_5.sqlite` → linked rollout JSONL. (Earlier heuristic based on `logs_2.sqlite` max-rowid was discarded — Codex CLI writes heartbeat rows constantly, so it fired ~every 20 s falsely.)
- Speaker power-amp (AW8737A) is held low at boot and only enabled for the duration of each chime → no continuous hiss/ticking between events.

**Global button-B clear gesture**
- Single press: `{"type":"submit"}` (Enter) — delayed by 300 ms to allow a double-press to override it.
- Double press inside 300 ms: `{"type":"clear"}` → server calls `clear_input()` which runs Cmd+A then Backspace via AppleScript.

**No-sleep while plugged in**
- `battery_task` exposes `s_on_usb_power` (Vin ≥ 4500 mV).
- `maybe_enter_idle_sleep()` short-circuits when USB power is present; on battery the existing 2-minute timeout still applies.

#### Hardware enablement (audio path)

The board has an ES8311 codec + AW8737A 1 W speaker amp + built-in speaker — the previous firmware just never enabled the output path. Today's wiring:
- `IOE_SPK_PA = M5IOE1_PIN_10` and `SPK_PA_PIN = GPIO 14` are toggled together (both have to be high). They stay low at boot.
- `esp_codec_dev_set_out_vol(100)` + `esp_codec_dev_set_out_mute(false)` are called after `esp_codec_dev_open`.
- PA enable order matches the official UserDemo: codec opens first, then PA toggles on per-chime.

#### Partition table resize
- App partition slots grew from 3.5 MB to 5 MB each. `storage` (SPIFFS — never mounted by our firmware) shrank from 9.4 MB to ~6 MB to free the space.
- Required a full USB re-flash; NVS (Wi-Fi credentials, hostname) was preserved because its offset (0x9000) is unchanged.

#### Bug fixes / polish
- Chime detection initially used a Claude busy heuristic that flapped on every tool-result; replaced with the `stop_reason == end_turn` ISO-timestamp signal (strictly forward → no replay on session-switch).
- Dashboard layout tightened: brand row moved into the left column above the big "% REMAINING", right-column compact rows narrowed to clear the round-display bezel, ~40 px vertical gap between halves.
- Pet screen scale reduced from 2.5× to 2× to stop frame drops on the S3 (AA off, smaller render area).
- Wave animation frame count corrected from 6 → 4 (petdex sprites only have 4 non-empty frames in row 3; the extra two caused a blank-frame "flash" each loop).
- `version.txt` is now gitignored — it was being regenerated by CMake on every build and polluting diffs.

## <a id="中文"></a>2026-06-04 · 十字导航扩展 / 多宠物 / 分区扩容 / 任务完成提示音

### 重点
- 主界面（Claude / Codex 左右滑）基础上扩展成"十字"导航：上滑进入 **Dashboard**（数据总览），下滑进入 **Pet**（宠物展示，三只猫之间可左右滑切换）。任一辅助屏反向滑动回到刚才那个主页。
- **设备终于会发声了**：Claude Code 完成一次回答时发 880 Hz 单音；Codex 完成一次任务时发 660 Hz 单音。功放只在响铃期间通电，空闲时间完全静音。
- **全局右键手势**：单击 = Enter，双击 = Cmd+A + Delete（清空输入框）。
- **插着 USB 时不再自动休眠**，电池供电时保留原来 2 分钟无操作进入休眠的逻辑。

### 新增功能

**多宠物（kitty / diana / nier）**
- 在原本的小猫（petdex #77）之外加了 Diana（#diana-2）和 2B（#nier-2b）两只，均来自 petdex 开源 sprite 库。
- 宠物屏现在是左右滑切换的"宠物列表"：kitty → diana → nier，到头停住（不循环）。任一宠物屏上滑回到刚才那个主页（Claude / Codex 看你从哪个上来）。
- 用 `PetSpec` 表把宠物状态机做成了通用的 —— 加一只新宠物只需在 `PETS[]` 数组里多一行。
- 每只宠物用 96×104 的 RGB565A8 帧（从 petdex 原图 192×208 缩 2 倍），三只一起占 ~2.5 MB。
- **行为节奏**：默认连续播 2–5 个 idle 周期（约 5–13 秒），中间随机插一次表情动作（wait / fail / review），完了回 idle。录音键按住时强制切到 wave。
- 帧时长统一拉到 petdex 默认值的 1.5 倍，看起来不再"小跑"。

**任务完成提示音**
- 新增 `mic-test/main/chime.{h,cpp}` 模块：独立 FreeRTOS 任务从单元素队列里取事件，生成带快速指数衰减 + ~6ms attack ramp 的正弦波样本，按满音量推到 ES8311。
- Mac 端 `server.py` 的 `push_state_loop` 在检测到 **原生的任务完成信号**时发一帧 WS：`{"type":"chime","agent":"claude"|"codex"}`。
  - **Claude**：扫 `~/.claude/projects/*/*.jsonl` 的最近 assistant 消息，找 `stop_reason == "end_turn"` 的时间戳，**只在时间戳严格往前走时**才响（解决了之前在多个 session 之间跳来跳去导致反复响的 bug）。
  - **Codex**：从 `~/.codex/state_5.sqlite` 找最近 thread → 对应 rollout JSONL → 倒着找 `event_msg.type == "task_complete"` 事件。同样要时间戳严格前进才响。（一开始用 `logs_2.sqlite` 的 max-rowid 反推 busy，被证明太脏 —— Codex CLI 有后台心跳日志，每 20 秒乱触发一次，已经丢弃。）
- 喇叭功放（AW8737A）开机后保持关闭，只在响铃时短暂通电 → 平时设备绝对安静，不再有那种"嗒嗒嗒"的底噪。

**右键双击清空**
- 单击：300 ms 后发 `{"type":"submit"}` —— 留时间窗等可能的双击。
- 300 ms 内连按两次：发 `{"type":"clear"}` → server 调 `clear_input()` 用 AppleScript 发 Cmd+A 然后退格，清空当前焦点输入框。

**USB 供电时常亮**
- `battery_task` 把当前是否检测到充电（Vin ≥ 4500 mV）写入 `s_on_usb_power`。
- `maybe_enter_idle_sleep()` 看到这个标志就直接返回 → 永不进 idle sleep。拔掉 USB 后还是按原 2 分钟逻辑走。

### 硬件相关（音频通路启用）

板子本身有 ES8311 codec + AW8737A 1W 功放 + 内置喇叭 —— 之前固件压根没启用输出端。今天补齐：
- `IOE_SPK_PA = M5IOE1_PIN_10` 和直连 `GPIO 14` **两个引脚都得拉高**（缺一不可），开机时保持低。
- `esp_codec_dev_open` 之后立刻调 `set_out_vol(100)` + `set_out_mute(false)`。
- PA 启用顺序对齐官方 UserDemo：先 codec open、然后 chime 任务来一次响一次。

### 分区表扩容
- 单个 app 槽从 3.5 MB 扩到 5 MB；`storage` 分区（SPIFFS，固件从来没挂载过）从 9.4 MB 缩到 ~6 MB 把空间让出来。
- 改完需要 USB 整片重烧一次，但 NVS 偏移（0x9000）不变 → Wi-Fi 配置和 hostname 不会丢。

### Bug fix / 打磨
- chime 检测一开始用 Claude busy 启发式，每次 tool_result 都会触发；换成 `stop_reason == end_turn` ISO 时间戳信号，再加上"严格前进"判定，多 session 切换也不会重放历史完成。
- Dashboard 布局收紧：brand 行挪到大百分比左列上方组成一列，右列收窄避免出圆，上下两块间距大约 40 px。
- 宠物屏放大倍数从 2.5× 降到 2×（关掉 AA，渲染面积砍 36%），ESP32-S3 不再掉帧。
- wave 行帧数从 6 改成 4（petdex 这一行实际只有 4 帧有内容，多提取 2 帧造成每个循环出现一次 "空白帧闪烁"）。
- `version.txt` 加入 gitignore —— CMake 每次构建都重生成，之前一直污染 diff。
